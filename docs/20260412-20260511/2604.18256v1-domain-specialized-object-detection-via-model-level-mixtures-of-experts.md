## 摘要
专家混合（Mixture-of-Experts，MoE）模型为组合专门化的神经网络提供了结构化的方法，并比传统的集成模型具有更高的可解释性。虽然 MoE 已成功应用于图像分类和语义分割任务，但由于在融合密集且结构化的预测时存在挑战，其在目标检测中的应用仍然有限。在本研究中，我们探讨了模型级的目标检测器混合体系，并分析其在提升目标检测性能与可解释性方面的适用性。我们提出了一种 MoE 架构，该架构结合了基于 YOLO 的检测器，这些检测器在语义上不重叠的数据子集上进行训练，并配备一个学习型的门控网络以动态加权专家的贡献。我们研究了不同的检测结果融合策略以及门控机制的训练方法，包括通过平衡损失来防止专家坍缩的策略。在 BDD100K 数据集上的实验表明，所提出的 MoE 模型在性能上始终优于标准集成方法，并提供了跨领域专家专门化的洞察，凸显出模型级 MoE 是目标检测中传统集成的一种可行替代方案。我们的代码可在 https://github.com/KASTEL-MobilityLab/mixtures-of-experts/ 获取。

## Abstract
Mixture-of-Experts (MoE) models provide a structured approach to combining specialized neural networks and offer greater interpretability than conventional ensembles. While MoEs have been successfully applied to image classification and semantic segmentation, their use in object detection remains limited due to challenges in merging dense and structured predictions. In this work, we investigate model-level mixtures of object detectors and analyze their suitability for improving performance and interpretability in object detection. We propose an MoE architecture that combines YOLO-based detectors trained on semantically disjoint data subsets, with a learned gating network that dynamically weights expert contributions. We study different strategies for fusing detection outputs and for training the gating mechanism, including balancing losses to prevent expert collapse. Experiments on the BDD100K dataset demonstrate that the proposed MoE consistently outperforms standard ensemble approaches and provides insights into expert specialization across domains, highlighting model-level MoEs as a viable alternative to traditional ensembling for object detection. Our code is available at https://github.com/KASTEL-MobilityLab/mixtures-of-experts/.

## 一、研究背景与核心问题

- **研究动机**：  
  对象检测在实际场景（如自动驾驶、安防监控等）中要求高精度与高可解释性。然而，传统的检测模型集成（ensemble）虽然能提升性能，但可解释性较差，难以理解单个检测器的作用。  
  Mixture-of-Experts（MoE）模型通过显式的专家路由机制（gating）提供一种结构化的模型整合方式，已在分类和分割任务中取得成功，但在**目标检测**中面临预测结构复杂（边界框 + 类别概率）导致的融合困难。  

- **核心问题**：  
  如何设计一种**模型级专家混合（Model-level MoE）架构**，既能提高检测性能，又能通过显式的专家权重分配提升模型的可解释性。

## 二、方法论：架构与技术细节

### 1. 核心思想
- 将多个基于 **YOLO** 的对象检测器视为 “专家”。
- 各专家在 **语义上不重叠的数据子集**（如白天 / 夜晚）上独立训练。
- 使用一个**可学习的门控网络（Gating Network）**，根据输入图像特征动态预测每个专家的权重，以实现专家融合。

### 2. 模型框架
- **专家集合**：包含 n 个 YOLO-based 检测器 \( F_i^{expert} \)。
- **门控网络输入**：各专家选定层的特征图，经拼接后输入至 Gate。
- **门控网络输出**：一组权重 \( w_i^{expert} \)，通过 softmax 正归一化，控制每个专家的贡献。
- **加权融合公式**：  
  \[
  \tilde y_i^{expert} = w_i^{expert} \cdot y_i^{expert}
  \]
  其中 \( y_i^{expert} \)为专家的原始输出（含边界框与分类分数），且加权发生在 **解码前**，保证定位与分类一致性。
- **后处理**：统一采用非极大值抑制（NMS）或加权抑制（如 NMW）来得到最终检测结果。

### 3. 门控网络训练策略
- **冻结专家权重，仅训练 Gate**。
- **防止专家坍缩（collapse）**：加入平衡损失，使 Gate 不偏向单个专家。
  - 评估的平衡损失包括：
    - Importance loss（均衡专家重要性分布）
    - KL 散度损失（与均匀分布距离最小化）
    - Entropy loss（最大化分配熵）
  - 最有效的是**样本级熵正则项（sample-wise entropy regularization）**。
- **Domain-aware gate training**：在门控输入含域标签（如白天 / 夜晚）时，用交叉熵监督训练路由机制，但此方式实验效果略差于无监督版本。

## 三、实验设计

### 1. 数据集与场景
- **BDD100K**（Berkeley DeepDrive）：自动驾驶场景数据集。
- 按时间元数据参数 `timeofday` 分为：
  - Daytime（白天）
  - Nighttime（夜晚）
  - Dawn/Dusk（黎明/黄昏）
  - Undefined（未定义）
- 白天与夜晚数据用于训练专家模型；黎明/黄昏与未定义数据作为 **跨域测试（OOD）**。

### 2. 对比方法
- **单模型基线**（YOLOv7、YOLOv9等）
- **传统集成方法**：如 Weighted Boxes Fusion (WBF)、Non-Maximum Weighted (NMW)、Soft-NMS。
- **所提 MoE 模型**：具有动态门控与加权预测。

### 3. 评测指标
- 使用 **mAP@0.5**（平均精度），并在各子域分别评测性能。

## 六、主要结论与发现

1. **性能表现**：  
   - MoE 在白天与夜晚子域上的 mAP 明显高于固定集成方法（最大提升约 0.15 相对比）。
   - 对 OOD 数据（dawn/dusk、undefined），MoE 与 ensemble 性能相当或略优。
   - 虽未超越在全数据集训练的单一大型模型，但在专域任务上表现最佳。

2. **可解释性提升**：  
   - 路由分析显示 Gate 自动学习域相关模式：白天图像更依赖 daytime expert，夜晚反之。
   - 分歧分析揭示专家间的互补检测行为（部分目标由单一专家独立侦测）。

3. **门控机制效果**：  
   - 样本级熵平衡最稳定；避免专家坍缩；
   - 简单卷积 + FC 两层的门控架构在性能与复杂度间取得平衡。

## 八、不足与局限

- **数据覆盖局限**：仅在 BDD100K 单一数据集验证，缺乏跨领域的扩展性评估。
- **性能上限**：MoE 未超越最强的单模型基线（全数据训练），说明门控机制尚未完全优化。
- **复杂度与可扩展性**：MoE 推理时间与参数量约为单模型的两倍，实际部署需权衡计算成本。
- **域标签依赖风险**：若领域划分不明显或存在混合特征，门控决策可能不稳定。
- **可解释性主观性**：虽显示路由分布，但仍需更可量化的指标来评估解释性。
